## Overview

Health Samurai builds Aidbox — an FHIR backend used across five product surfaces (admin console, developer portal, FireCamp event app, internal tooling, marketing). One design system had to cover them all without slowing the engineering cadence.

## What I owned

- shadcn-based component library with FHIR-aware variants (resource pickers, reference fields, value-set autocompletes)
- Figma ↔ code bidirectional sync through Code Connect — designers and engineers see the same prop names, same defaults
- Storybook governance: every component lands with stories, every story is the source of truth for the design file
- A Claude skill that pulls Figma components into Storybook scaffolds when a designer wants something tweaked but a dev isn't free
- Component coverage decisions: when shadcn ships it, we use it; when it doesn't, we wrap and contribute back
